e) As instruçôes atômicas não-triviais (por exemplo, um LOAD ou um STOR são instruções "triviais" por fazerem apenas uma coisa) que estão disponíveis e o que elas fazem:

BTC - Bit Test and Complement: Seleciona um bit de uma bit string (primeiro operando) na posição indicada (segundo operando) e o armazena na flag CF, em seguinda faz substitui o valor do bit indicado na bit string pelo seu complemento.

BTR - Bit Test and Reset: Seleciona um bit de uma bit string (primeiro operando) na posição indicada (segundo operando) e o armazena na flag CF, em seguinda faz altera o valor do bit indicado na bit string para 0.

BTS - Bit Test and Set: Seleciona um bit de uma bit string (primeiro operando) na posição indicada (segundo operando) e o armazena na flag CF, em seguinda faz altera o valor do bit indicado na bit string para 1.

CMPXCHG - Compare and Exchange: Compara o valor do registrador (AL, AX, EAX, ou RAX) com o primeiro operando (destino). Se os valores forem iguais, o segundo operando (fonte) é carregado no operando destino. Caso contrário, o operando destino é carregado no registrador (AL, AX, EAX, ou RAX).

XADD - Exchange and Add: Troca o primeiro operando (operando destino) com o valor do segundo operando (operando fonte), então carrega a soma dos dois valores para o operando destino


f) A velocidade da memória que está rodando na sua máquina. Ela é rápida o suficiente para o processador? Sim? Não? Por quê?

Memory Types:		DDR3-1333 
Max Memory Bandwidth: 	21.3 GB/s => 21504 MB/s
Dual-Channel Mode
memória: 1333mhz x 128 (dual channel) /8 = 21328 MB/s
processador: 2.8 ghz => 2800 mhz x 64 / 8 = 22400 MB/s

Sim, ela é rapida o suficiente para o processador. Pois a velocidade da memória é quase equivalente a do processador, assim a memória tem velocidade suficiente para acompanhar a velocidade (de troca de dados) do processador.
